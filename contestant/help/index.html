Different Kinds of Contests

  • help/formal_contest: About formal contests, in which you have to participate at a designated time. You are given problems only on paper, at the start of the contest. Judging is usually manual, though an autojudge may be used to give quick preliminary estimates of submission scores.
  • help/informal_contest: About informal contests, in which you may participate on your own time schedule. There is an operation to get a problem, and the problem time is between when you get the problem and when you submit its solution; or when your team gets its FIRST problem, and when each problem's solution is submitted. Judging is done by an autojudge.
  • help/untimed_contest: About untimed contests, in which you have as much time as you like to do a problem, without the time affecting your score. The contest is usually scored according to how you make submissions of incorrect solutions. You may submit so as to get back the judge's input for the first test case that fails, or to get back the judge's input and judge's output for that test case. The more such feedback you ask for, the greater is the penalty for an incorrect submission. Judging is done by an autojudge.

Different Kinds of Accounts

  • Contest Provided Accounts

    You are provided an account by the managers of the contest. For formal contests the managers log in for you. For other contests, you are given an account name and password, and log in remotely using ssh.

    • help/solving: How to write a solution file, how to test it, how to submit it, when you are using an account given you.
    • help/xterminals: How to make windows if you are using an xterminal emulator.

  • Personal Accounts

    You use your own personal account to solve problems, and email submissions and requests to the autojudge.

    • help/email_solving: How to write a solution file, how to test it, how to submit it, when you are using your own personal account.
    • Using UNIX tools to make your account look like an account provided by the contest managers.
