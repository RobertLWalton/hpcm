HPCM Help Page

Different Kinds of Contests

  • help/email_contest: About email contests, in which you provide our OWN account and email solutions to an autojudge. There is an operation (hpcm_get) to get a problem, and the problem time is between when you get the problem and when you submit its solution. See below for more details.
  • help/informal_contest: About informal contests, in which an account is given to you but you may participate in the contest on your own time schedule. There is an operation (hpcm_get) to get a problem, and the problem time is between when you get the problem and when you submit its solution; or when your team gets its FIRST problem, and when each problem's solution is submitted. Judging is usually done by an autojudge. See below for more information about terminal setup.
  • help/formal_contest: About formal contests, in which an account is given to you and you have to participate at a designated time. You are given problems only on paper, at the start of the contest. Judging is usually manual, though an autojudge may be used to give quick preliminary estimates of submission scores. See below for more information about terminal setup.

Extra Info for Email Contests

  • help/email_unix_tools: Available tools for email contest contestants running UNIX.

    Code for tools written in Bourne shell (e.g. bash) and procmail:

    • help/sh/hpcm_get Send email to the judge to get files.
    • help/sh/hpcm_extract Extract the files from the replies.
    • help/sh/hpcm_sandbox Execute programs with resource limits.
    • help/sh/hpcm_submit Send submission file to the judge.
    • help/sh/hpcm_sendmail Send email to the judge.
    • help/sh/procmailrc Inclusion for .procmailrc to invoke hpcm_extract automatically to extract files gotten by hpcm_get.

    Code for tools written in TCL (tclsh):

    • help/tcl/hpcm_get Send email to the judge to get files.
    • help/tcl/hpcm_extract Extract the files from the replies.
